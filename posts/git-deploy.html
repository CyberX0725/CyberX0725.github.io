使用 GitHub Pages 部署静态网站完整指南

2026.07.05 10 分钟阅读 CyberX0725

GitHub Pages 是 GitHub 提供的免费静态网站托管服务，非常适合部署个人博客、项目文档或作品集。本指南将从零开始，带你完成完整的部署流程。

什么是 GitHub Pages？

GitHub Pages 允许你直接从 GitHub 仓库托管静态网站，无需服务器、无需数据库、无需付费。每个 GitHub 账户可以拥有一个用户站点和无限个项目站点。

用户站点访问地址格式：https://<username>.github.io
项目站点访问地址格式：https://<username>.github.io/<repository>

第一步：创建仓库

对于用户站点，仓库必须命名为 <username>.github.io。例如，我的用户名是 CyberX0725，所以仓库名为 CyberX0725.github.io。

  1. 登录 GitHub，点击右上角 + → New repository
  2. 仓库名填入 CyberX0725.github.io（替换为你的用户名）
  3. 选择 Public（公开）
  4. 勾选 Add a README file
  5. 点击 Create repository

第二步：初始化本地项目

将远程仓库克隆到本地，然后添加你的网站文件：

# 克隆仓库
git clone https://github.com/CyberX0725/CyberX0725.github.io.git

# 进入目录
cd CyberX0725.github.io

# 创建项目结构
mkdir -p assets/css assets/js assets/images posts

# 添加你的 HTML、CSS、JS 文件
# ...

一个典型的项目结构如下：

CyberX0725.github.io/
├── index.html
├── about.html
├── archive.html
├── 404.html
├── assets/
│   ├── css/
│   │   └── style.css
│   ├── js/
│   │   └── main.js
│   └── images/
│       └── avatar.svg
├── posts/
│   ├── hello-world.html
│   └── ...
└── README.md

第三步：推送到 GitHub

将文件添加到 Git 并推送到远程仓库：

# 添加所有文件
git add .

# 提交更改
git commit -m "Initial blog setup"

# 推送到 GitHub
git push origin main
注意：如果你的默认分支是 master 而非 main，请相应调整命令。GitHub 现在默认使用 main 作为主分支名。

第四步：启用 GitHub Pages

对于用户站点（<username>.github.io 仓库），推送代码后 GitHub Pages 会自动启用。对于项目站点，需要手动开启：

  1. 进入仓库的 Settings 选项卡
  2. 在左侧菜单找到 Pages
  3. 在 Source 下拉菜单中选择分支（通常是 main）
  4. 选择根目录 / (root)
  5. 点击 Save

等待 1-2 分钟，你的网站就会在 https://cyberx0725.github.io 上线。

第五步：配置 Git 用户信息

如果你是第一次使用 Git，需要配置用户信息：

git config --global user.name "CyberX0725"
git config --global user.email "3262525349@qq.com"

进阶：配置自定义域名

如果你有自己的域名，可以配置为 GitHub Pages 的自定义域名：

  1. 在仓库根目录创建 CNAME 文件（无扩展名）
  2. 文件内容只写你的域名，例如 blog.cyberx0725.com
  3. 在你的域名 DNS 设置中添加 CNAME 记录，指向 cyberx0725.github.io
  4. 等待 DNS 生效（可能需要几小时）
# CNAME 文件内容
blog.cyberx0725.com

进阶：使用 GitHub Actions 自动部署

如果你使用构建工具（如 Vite、Webpack），可以配置 GitHub Actions 自动构建并部署：

# .github/workflows/deploy.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Setup Node.js
        uses: actions/setup-node@v3
        with:
          node-version: '18'
      - name: Install dependencies
        run: npm ci
      - name: Build
        run: npm run build
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          github_token: ${{ secrets.GITHUB_TOKEN }}
          publish_dir: ./dist

常见问题

网站显示 404？

  • 确认仓库名正确：<username>.github.io
  • 确认仓库是 Public（免费账户仅支持公开仓库）
  • 确认 index.html 在仓库根目录
  • 等待 1-2 分钟让部署完成

资源加载失败（404）？

检查文件路径。对于项目站点（非用户站点），资源路径需要包含仓库名：

<!-- 用户站点可用相对路径 -->
<link rel="stylesheet" href="assets/css/style.css">

<!-- 项目站点需要绝对路径 -->
<link rel="stylesheet" href="/repository-name/assets/css/style.css">

HTTPS 证书未生效？

在 Settings → Pages 中勾选 Enforce HTTPS。首次启用可能需要等待几分钟到几小时。

总结

GitHub Pages 是部署静态网站的绝佳选择——免费、简单、与 Git 工作流无缝集成。无论是个人博客、项目文档还是在线简历，都可以在几分钟内上线。

最好的部署方式是最简单的方式。不要为了部署而部署，把精力放在内容上。
